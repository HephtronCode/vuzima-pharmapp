# Branching and Commit Conventions

Last updated: 2026-04-02
Owner: Engineering

## Branching

- Feature branches per unit of work.
- Keep branch names clear (for example `feat/staff-enable-action`).

## Commit style

- Prefer concise imperative messages.
- Focus on intent and outcome, not just file changes.

Examples:

- `add admin enable action for disabled staff accounts`
- `improve inventory csv import guidance and validation messages`

## PR expectations

- Include summary, rationale, and test notes.
- Mention docs updated.
